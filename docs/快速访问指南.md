## 🚀 快速开始

### 方法1：同一局域网访问（最简单）

如果访问者和你在同一局域网（WiFi/路由器）内：

1. **确保服务器正在运行**
   ```bash
   # 如果还没启动，运行：
   ./start-remote.sh
   ```

2. **访问地址**
   - 前端：`http://192.168.110.15:5173`
   - 后端API：`http://192.168.110.15:8000`

3. **开放防火墙端口**（如果无法访问）
   ```bash
   # Ubuntu/Debian
   sudo ufw allow 8000/tcp
   sudo ufw allow 5173/tcp
   
   # CentOS/RHEL
   sudo firewall-cmd --permanent --add-port=8000/tcp
   sudo firewall-cmd --permanent --add-port=5173/tcp
   sudo firewall-cmd --reload
   ```

### 方法2：公网访问（需要配置）

#### 选项A：路由器端口转发

1. **登录路由器管理界面**（通常是 `192.168.1.1` 或 `192.168.0.1`）
2. **找到端口转发/虚拟服务器设置**
3. **添加两条规则**：
   - 外网端口 `5173` → 内网 `192.168.110.15:5173`
   - 外网端口 `8000` → 内网 `192.168.110.15:8000`
4. **访问地址**：`http://117.176.159.226:5173`

#### 选项B：使用内网穿透（推荐，无需路由器配置）

**使用 ngrok（最简单）**：

```bash
# 1. 安装 ngrok
sudo snap install ngrok
# 或下载：https://ngrok.com/download

# 2. 注册并获取 token
# 访问：https://dashboard.ngrok.com/get-started/your-authtoken
ngrok config add-authtoken YOUR_TOKEN

# 3. 启动前端隧道
ngrok http 5173

# 4. 启动后端隧道（新终端窗口）
ngrok http 8000
```

ngrok 会提供类似 `https://xxxx-xx-xx-xx-xx.ngrok-free.app` 的公网地址。

## 🔧 故障排查

### 无法访问？

1. **检查服务器是否运行**
   ```bash
   netstat -tlnp | grep -E ':(8000|5173)'
   ```

2. **检查防火墙**
   ```bash
   sudo ufw status
   # 或
   sudo firewall-cmd --list-all
   ```

3. **检查IP地址**
   ```bash
   hostname -I
   ```

4. **测试本地访问**
   ```bash
   curl http://localhost:5173
   curl http://localhost:8000/api/admin/v1/auth/me
   ```

### 前端能访问但API报错？

检查 `vite.config.ts` 中的 proxy 配置，确保指向正确的后端地址。
